# Actionable Finance Agent

금융 정보 수집, 분석, 암호화폐 매매를 수행하는 에이전트 시스템.

## 서브에이전트

### coin-trading

업비트/바이낸스 거래소를 통한 암호화폐 매매 에이전트.

**실행 조건:**
- 코인/암호화폐 시세 조회 요청
- 업비트/바이낸스 매매 요청 (매수, 매도)
- 잔고/포트폴리오 확인 요청
- 마진 LTV, 청산가 조회 요청

**거래소 선택:**
- "업비트", "upbit" → 업비트
- "바이낸스", "binance" → 바이낸스
- 명시 없으면 → 사용자에게 확인

**지원 기능:**

| 분류 | 업비트 | 바이낸스 |
|------|--------|----------|
| 시세 조회 | 현재가, 호가창, 캔들, 마켓 목록 | 현재가, 호가창, 캔들, 마켓 목록 |
| 자산 관리 | 잔고 조회, 수익률 | Spot/Margin 잔고, 수익률 |
| 매매 | 시장가/지정가 매수/매도 | Spot/Margin 시장가/지정가 |
| 마진 | - | LTV 계산, 청산가 시뮬레이션, 대출/상환 |

**참조 스킬:** `upbit-trading`, `binance-trading`

---

### stock-trading

한국투자증권을 통한 국내 주식 매매 에이전트.

**실행 조건:**
- 주식 시세 조회 요청 (삼성전자, 005930 등)
- 주식 매매 요청 (매수, 매도)
- 주식 잔고/수익률 확인 요청
- "한투", "주식" 키워드 포함 시

**지원 기능:**

| 분류 | 기능 |
|------|------|
| 시세 조회 | 현재가, 호가창, 일봉/분봉, 종목 검색 |
| 자산 관리 | 보유종목 조회, 수익률 확인 |
| 매매 | 시장가/지정가 매수/매도, 주문 취소 |

**참조 스킬:** `kis-trading`

### daily-summary

텔레그램 메시지 수집 및 요약 전송 에이전트.

**실행 조건:**
- "일일 요약", "daily summary", "뉴스 요약" 요청
- 정기적인 요약 전송 필요 시

**워크플로우:**
1. 텔레그램 메시지 수집 (최근 24시간)
2. 카테고리별 분석 (시장 동향, 뉴스, 투자 정보)
3. 요약 생성
4. 텔레그램 전송

**참조 스킬:** `telegram-collector`, `daily-summary`

### scheduler-manager

스케줄러 작업을 관리하는 에이전트.

**실행 조건:**
- "X시에 Y해줘" 형태의 시간 기반 요청
- "매일", "매주", "정기적으로" 등의 반복 요청
- 스케줄 목록 조회, 추가, 삭제 요청

**중요 - 일회성 vs 반복:**
- "4시에 ~해줘" → **일회성** (once)
- "오늘/내일 4시에 ~해줘" → **일회성** (once)
- "**매일** 4시에 ~해줘" → 반복 (cron)
- "**평일/주말** 4시에 ~해줘" → 반복 (cron)
- ⚡ **기본값: 일회성** - 반복 키워드 없으면 일회성으로 처리

**오전/오후 해석:**
- 1~6시 → 새벽 (01~06)
- 7~11시 → 오전 (07~11)
- 1~6시 (낮 대화 중) → 오후 (13~18)
- 7~11시 (저녁 대화 중) → 저녁 (19~23)
- **모호한 경우 반드시 확인**: "오전 3시인가요, 오후 3시인가요?"

**참조 스킬:** `scheduler`

## 매매 안전 규칙

1. **사전 확인 필수**: 모든 매매 주문 전 사용자 확인
2. **정보 제공**: 현재가, 예상 체결 수량/금액 안내
3. **최소 금액**:
   - 업비트: KRW 마켓 기준 5,000원 이상
   - 바이낸스: 보통 10 USDT 이상
4. **잔고 검증**: 주문 전 충분한 잔고 확인
5. **마진 거래 주의**:
   - LTV 확인 후 진행
   - Margin Level 1.5 미만 시 경고
   - 청산 위험 안내

## 사용자 응답 (텔레그램)

사용자에게 응답할 때는 텔레그램으로 전송한다.

```bash
uv run python scripts/send_telegram.py "응답 메시지"
```

**응답 가이드라인:**
- 2000자 이내 유지
- 이모지 활용하여 가독성 향상
- 핵심 정보 먼저, 상세 내용은 후순위
- 시세 정보는 간결하게
- 추세 분석은 데이터 기반 객관적으로
- 매매 결과는 주문번호, 체결 내역 포함
- `---` 구분선은 최대 5개까지만 (사용자가 모바일에서 확인)
